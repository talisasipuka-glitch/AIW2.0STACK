# Website Factory, Structure Overview

## What the Factory does

The factory runs a deterministic per-client pipeline (Stages 1 to 13).
Each client run clones the active niche template, overlays per-client
brand-dna, copy, and assets, builds the site with Vite + React + Tailwind,
runs design/build/SOP/performance QA, deploys to Vercel, and generates a
delivery report plus a client proposal.

## Per-niche template

The factory does NOT carry a baseline template. Module 2D generates a
per-niche template at `templates/{niche-slug}/` from captured niche
research (top-of-niche sites, design tokens, niche playbook, niche-specific
QA checklists). Stage 10.1 clones the active niche template per client.

For this stack, the active niche is **personal-injury-lawyers**. The
template exists at `templates/personal-injury-lawyers/` (generated by
`/build-niche-template`, registered in `config/template-routes.json`,
`MANIFEST.json` present, all 6 validator gates passing).

If no per-niche template exists for the active niche, Stage 10.1 halts with
a clear error pointing back to `/build-niche-template`.

## Pipeline

| # | Stage | Output | Gate threshold |
|---|---|---|---|
| 1 | Intake | `intake-form.json` + folder scaffold | 4 fields validated (businessName, websiteUrl, phone, email) |
| 2 | Research | `research.json` + `brand-research.md` | 11 sections present |
| 3 | SEO | `seo-strategy.json` + audit + revenue calc | 7 audit sections, >=5 keyword gaps, $-figure |
| 4 | Asset scraping | `[Client] Assets/{logo,photos,badges,owner}/*` | logo found OR halt |
| 5 | Strategy | `strategy/strategy.json` + `sitemap.json` | per niche playbook |
| 6 | Copywriting | `copy-deck.json` + per-page `.md` | every sitemap page covered, zero placeholders |
| 7 | Brand DNA | `brand-dna.json` (5-pass extraction) | schema valid, aggregate confidence >= 0.70 |
| GATE 1 | Brand DNA approval | `/approve-brand-dna` | only if confidence < 0.70 |
| 9 | Hero image | `hero-final-{desktop,mobile}.png` | both pass validation |
| 10.1 | Build | per-client `Website/` + `dist/` | Vite builds, every section in the niche template's canonical order populates |
| 10.2 | Personalise | per-client overrides applied | personalisation checklist passed |
| 10.3 | Uplift (optional) | optional extras wired per brand-dna signals | no-op when no triggers fire |
| 10.4a | Design fidelity QA | `qa-screenshots/*` + `scores.json` | aggregate SSIM >= 0.90 per region, loop cap 5 |
| 10.4b | SOP QA | `sop-runs/run-N/*` | >= 95% pass, em-dash count = 0, loop cap 10 |
| 11 | Deploy | `vercel-url.txt` + `deploy-log.json` | Vercel build succeeds |
| 12 | Delivery | `delivery-report.md` | every checklist item passed |
| 13 | Proposal | `proposal.html` | zero `[BRACKET]` placeholders, iframe URL valid |

The student runs `/build-all` from inside the factory to kick off the whole
pipeline. The only approval gate is Stage 7 (brand-dna confidence < 0.70).

Failure protocol: any gate failure or loop-cap breach writes
`clients/[Client Name]/MANUAL-INTERVENTION-NEEDED.md`, sets
`pipeline-state.json status: "halted"`, logs to `build-log.md`, and halts.
No downstream stages run.

## Inputs the Factory needs (per client)

Each client lands in `clients/[Client Name]/`. The intake agent (Stage 1,
`00-intake.md`) validates 4 fields (businessName, websiteUrl, phone, email),
normalises phone/email, verifies the website is reachable, and scaffolds:

```
Pipeline Data/intake/
Pipeline Data/research/  (incl. website-capture/screenshots/)
Pipeline Data/strategy/
Pipeline Data/seo/
Pipeline Data/copy/
Pipeline Data/brand/
Pipeline Data/design/
Pipeline Data/hero-image/
Pipeline Data/logs/
Pipeline Data/deploy/
Pipeline Data/delivery/
[Client Name] Assets/{logo,badges,project-images,founder-photos}/
```

After intake, downstream stages populate:
- `Pipeline Data/research/research.json`
- `Pipeline Data/seo/audit-data.json`
- `Pipeline Data/strategy/strategy.json`
- `Pipeline Data/copy/copy-deck.md`
- `Pipeline Data/brand/brand-dna.json`
- `[Client Name] Assets/`, logo, photos, etc.

## Canonical brand-dna shape

Every component in every niche template reads from the canonical shape at
`references/brand-dna.shape.js`. Top-level keys:

`meta, company, contact, address, hours, businessHours, social, team,
team_group_photo, team_members, theme_mode, voice_register, shape_motif,
corner_overlay, palette, typography, reviews, services, serviceAreas,
trust_badges, press_logos, previous_projects, copy, process_steps,
why_choose_us, special_offers, faq, blog_posts, blog_categories,
location_pages, pages, credit`

`copy` is a nested bundle covering hero, trust chips/claims, form copy,
per-section label/heading/body text (topBar, blog, cta, faq, founder,
gallery, offers, process, reviews, serviceAreaCard, serviceAreas, services,
whyChoose). `pages` holds per-route copy bundles (about, serviceAreas,
locationDetail, blogPost, blog, contact, services, financing); empty `{}`
entries mean that page doesn't ship for the active niche.

Per-client values are written by Stage 7 (`brand-dna-agent`) into
`Pipeline Data/brand/brand-dna.json`. Stage 10.1 fills the niche template's
`src/config/brand-dna.js` from those values.

## What the per-niche template + playbook supply

`templates/personal-injury-lawyers/` carries:
- React components (`src/components/`) and pages (`src/pages/`), wired in
  `App.jsx`
- Tailwind config + foundation tokens (palette, typography, motion, shape
  motif) substituted from `niche-design-tokens.json`
- Niche-specific QA checklists at `.claude/checklists/`
  (`design-fidelity.md`, `sop-compliance.md`)
- Niche-specific SOPs and agents at `.claude/sops/`, `.claude/agents/`
- The niche playbook at `niche-playbook/`:
  - JSON: `asset-patterns.json`, `copy-locks.json`, `hero-mood-mapping.json`,
    `motion-preset.json`, `photo-manifest.json`, `process.json`,
    `proposal-pages.json`, `resonance-queries.json`, `theme.json`,
    `trust-signals.json`, `vocabulary.json`
  - Markdown contracts: `copy-blocklist-additions.md`, `copywriting.md`,
    `cro-rules.md`, `design-vocabulary.md`, `hero-composition.md`,
    `quantified-trust-templates.md`, plus `sop-overrides/00-master.md`
  - `trust-badges/` (curated SVGs or `MANUAL-DROP-NEEDED.md`)

The factory pipeline reads from the playbook for any niche-specific
decision and from the niche template for any structural decision.

## What the Factory defaults handle (universal)

- Theme mode logic (light/dark from logo brightness)
- Build stack (Vite + React + Tailwind) per the niche template's
  `package.json`
- SEO injection, schema markup
- Lighthouse perf gate (LCP < 3s, desktop and mobile)
- Vercel deploy
- Proposal HTML generation
- Universal AI-vocab blocklist (`references/copy/ai-vocab-blocklist.md`) +
  typographic standards (`references/copy/typographic-standards.md`)
- Universal invariants enforced at every stage: zero em-dashes, smart-quote
  enforcement, `prefers-reduced-motion: reduce` honoured, schema validity at
  every gate that writes JSON, logo present (HALT on missing)

Niche-specific values read from `niche-playbook/`:
- Locked CTA, form header, privacy line, mobile call button: from
  `copy-locks.json`
- Trust signal claim/badge counts + placements: from `trust-signals.json`
- Theme support (light/dark/both): from `theme.json`
  (personal-injury-lawyers is `light` only)
- Process section step count: from `process.json` (locked at 3 for this
  niche: Submit / We Investigate / We Fight)
- Home page section order: from the niche template's `HomePage.jsx`

## Outputs the Factory produces

- `clients/[Client Name]/[Client Name] Website/dist/`, built site
- Vercel deploy URL
- `clients/[Client Name]/[Client Name] Proposal/proposal.html` + live
  proposal URL
- Delivery report at `clients/[Client Name]/Pipeline Data/delivery/delivery-report.md`

## Handoff format

The Module 5 brief lands at `research/output/website-factory-brief.md`.
`/tailor-factory` reads it and updates any factory-wide settings that need
to align with the niche (e.g. agency branding in the proposal template).
