## Steps

### Step 0, Read accumulated lessons (universal)

Read `.claude/lessons/by-agent/02-seo-audit.md` and any per-client lessons
at `clients/[Client Name]/Pipeline Data/lessons/notes.md`. Apply every rule
as an override to the spec below.

### Step 1, Build the keyword clusters

Follow `.claude/sops/02-seo-audit.sop.md` Procedure steps 1 to 3: build the
primary `"[case type] lawyer [city]"` cluster for each covered case type,
the broader `"personal injury lawyer [city]"` / "near me" cluster, and the
long-tail clusters (free consultation, no win no fee, injury-type
combinations).

### Step 2, Confirm title/H1 formulas and schema plan

Follow steps 4 to 8: confirm the homepage title/H1 formula
(`"[Firm Name]: Personal Injury Lawyers [City/State]"`) and the case-type
page formula (`"[Case Type] Lawyer in [City] | [Firm Name]"`). Plan the full
schema set (Attorney, LegalService, LocalBusiness, AggregateRating/Review,
FAQPage, BreadcrumbList) with the brand-dna fields each will read. Identify
the PAA-driven homepage FAQ set and at least one case-type-specific FAQ per
case type.

## Pass criteria

Per `.claude/sops/02-seo-audit.sop.md` Pass criteria: keyword list covers
all covered case types plus broader/long-tail clusters; title/H1 formulas
confirmed for homepage and all case-type pages; full schema plan documented;
at least 5 homepage PAA questions and 1+ case-type FAQ per case type.

## When this agent halts

Per `.claude/sops/02-seo-audit.sop.md` When this stage halts: fewer than 2
case types covered (escalate for direction decision), or city/location
ambiguous (confirm primary target city).

## Outputs

title/H1 formulas, schema plan, and FAQ question sets. Hands off to
`04-strategy.md`.
