# Agent: copy-deck-agent (Stage 6)

## Role

Write the complete site copy in the serious, professional voice this niche
requires: hero, process steps, stats, trust bullets, all case-type pages,
founder story, testimonials, FAQ, and CTAs, honoring the 4-field LeadForm
cap and trust-claim placement rules.

## Prerequisites

Stage 5 (`04-strategy.md`) has completed and
`clients/[Client Name]/Pipeline Data/strategy/` exists.

## Inputs

- `clients/[Client Name]/Pipeline Data/strategy/`
- `clients/[Client Name]/Pipeline Data/research/`
- `copywriting/SKILL.md`
- `references/copy/ai-vocab-blocklist.md`
- `references/copy/typographic-standards.md`
- `.claude/sops/05-copy-deck.sop.md`

## Steps

### Step 0, Read accumulated lessons (universal)

Read `.claude/lessons/by-agent/05-copy-deck.md` and any per-client lessons
at `clients/[Client Name]/Pipeline Data/lessons/notes.md`. Apply every rule
as an override to the spec below.

### Step 1, Write hero, process, and trust copy

Follow `.claude/sops/05-copy-deck.sop.md` Procedure steps 1 to 5: hero H1/
subheadline/trust chips, 4-field LeadForm copy (name, phone, email, case
description, no implied extra fields), `process_steps[]`, StatsBar copy
with plain numerals for `tabular-nums`, and `why_choose_us[]`.

### Step 2, Write case-type pages, founder story, testimonials, FAQ

Follow steps 6 to 12: `pages.services.items[]` for each covered case type
(name, description, settlementFigure if disclosable, body, faq[]);
FounderStory 2-paragraph specific personal narrative; `reviews.items[]`
testimonial summaries tied to case type and outcome; homepage `faq[]`
(PAA-driven); `copy.trustClaims[]`; final `copy.cta.*`; footer copy
including `company.licenseNumber`.

### Step 3, Run copy-lint

Follow step 13: run `tools/copy-lint.py --check` against
`references/copy/ai-vocab-blocklist.md` and
`references/copy/typographic-standards.md`. Fix any em-dash, blocklist, or
quote/tabular-nums findings before proceeding.

## Pass criteria

Per `.claude/sops/05-copy-deck.sop.md` Pass criteria: every brand-dna copy
path populated; LeadForm copy implies exactly 4 fields everywhere; all
covered case-type pages have name/description/body/faq[]; contingency
statement near hero CTA, footer CTA, every case-type RiskRemovalBand, and
final CTABand; "Available 24/7" copy only if confirmed; copy-lint passes;
FounderStory is specific, not generic.

## When this agent halts

Per `.claude/sops/05-copy-deck.sop.md` When this stage halts: copy-lint
fails (rewrite and re-run); a case-type page entry has no `body` or `faq[]`;
hero/footer LeadForm copy implies more than 4 fields.

## Outputs

`clients/[Client Name]/Pipeline Data/copy/` populated with the full copy
deck, copy-lint-clean. Hands off to `brand-dna-agent.md`.
