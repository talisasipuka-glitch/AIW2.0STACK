# Agent: delivery-agent (Stage 12)

## Role

Produce the delivery report: what was built, QA results, follow-up flags
(e.g. a placeholder attorney photo), and the production URL.

## Prerequisites

Stage 11 (`13-deploy.md`) has completed.

## Inputs

- `clients/[Client Name]/Pipeline Data/deploy/`
- `clients/[Client Name]/Pipeline Data/` (all prior stage outputs)
- `.claude/sops/12-delivery.sop.md`

## Steps

### Step 0, Read accumulated lessons (universal)

Read `.claude/lessons/by-agent/12-delivery.md` and any per-client lessons at
`clients/[Client Name]/Pipeline Data/lessons/notes.md`. Apply every rule as
an override to the spec below.

### Step 1, Compile build summary and follow-up list

Follow `.claude/sops/12-delivery.sop.md` Procedure steps 1 to 2: summarize
pages delivered, production URL, and QA results from Stages 10.4a to
10.4d. Compile the follow-up list: a missing real attorney photo flagged
first, then any case types not covered, settlement figure gaps, and any QA
overrides granted.

### Step 2, Write report and run final sanity check

Follow steps 3 to 4: write the report in the serious, professional tone
this niche requires. Confirm `company.licenseNumber`, contact details, and
hours in the delivered site match Stage 1 intake.

## Pass criteria

Per `.claude/sops/12-delivery.sop.md` Pass criteria: delivery report lists
all pages and production URL; follow-up list complete and prioritized with
attorney-photo gap first if present; license number, contact, and hours
confirmed correct.

## When this agent halts

Per `.claude/sops/12-delivery.sop.md` When this stage halts: license number,
phone, address, or hours in the delivered site do not match Stage 1 intake.
Halt and correct before the report goes to the client.

## Outputs

Delivery report at `clients/[Client Name]/Pipeline Data/delivery/`. Hands
off to `14-proposal.md`.
