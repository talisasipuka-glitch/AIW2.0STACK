# 9 - Hero Image

Implements: Stage 9. Runs BEFORE Stage 10.1.

## Purpose

Generate or finalize the hero image composition: named attorney photo,
trust chips, and the 4-field LeadForm layout, in the restrained,
professional mood this niche requires.

## Inputs (per-client)

```
clients/[Client Name]/Pipeline Data/brand/brand-dna.json
clients/[Client Name]/[Client Name] Assets/
clients/[Client Name]/Pipeline Data/brand-resonance/ (if Stage 7.5 ran)
```

## Procedure

1. Check Stage 4 assets for a usable attorney headshot or team photo. If a
   real photo exists, this stage's job is composition and background
   treatment around that photo, NOT generating a synthetic attorney face.
2. If no real attorney photo exists (flagged at Stage 4), generate a
   placeholder composition that does not depict a specific invented person
   as "the attorney." Use a generic but professional office/consultation
   scene instead, and flag clearly in the delivery report (Stage 12) that a
   real attorney photo should replace this before the client treats the
   site as final.
3. Compose the hero per `09-wireframe.md`: split layout, copy + LeadForm
   left/center, attorney image right (stacks on mobile with image first).
   Trust chips render above the H1.
4. Apply the niche's restrained motion mood: subtle entrance animation
   only, `--motion-duration: 600ms`, `--motion-stagger: 90ms`, honoring
   `prefers-reduced-motion`. No energetic or bouncy motion treatments.
5. Apply the palette restraint rule: near-black/charcoal as the dominant
   hero background or text color, bright yellow accent used ONLY for the
   CTA button, the "OPEN 24/7" badge (if applicable), and any dollar figure
   shown in the hero. Do not let yellow become a large background field.
6. If a chevron corner-overlay accent is used in the hero background, apply
   it at low opacity (0.08) per `09-template-spec.md` Section 1.
7. Verify the composed hero, with the LeadForm, fits above the 900px
   desktop fold per `09-wireframe.md`'s `composition.above_fold` check, and
   that the form's submit button is visible without scrolling at 1440x900.

## Pass criteria

- Hero image composition matches the split layout (copy+form /
  attorney image) with correct mobile stacking order.
- A real attorney photo is used if available; if not, a generic
  professional scene is used and flagged for replacement, not a synthetic
  "fake attorney."
- Palette restraint holds: yellow only on CTA, badge, and dollar figures.
- Motion preset is restrained (600ms duration, 90ms stagger,
  reduced-motion respected).
- Above-the-fold composition check passes at 1440x900.

## When this stage halts

- This stage does not hard-halt the pipeline, but if no real attorney photo
  exists, it MUST write a flag to the delivery report (Stage 12) that
  carries through to the client-facing proposal (Stage 13) as a follow-up
  item.

## Niche-specific notes

For this niche, the following pattern applies (derived from
`research/02-niche-research/personal-injury-lawyers/09-template-spec.md` +
the niche playbook):

The hero composition is one of the highest-weighted SSIM regions in this
template (0.95 threshold, weight 0.20 per `09-wireframe.md`), reflecting
its role as the first impression for a visitor who is, per
`02-customer-voice.md`, likely injured, stressed, and comparing firms
within hours of an accident. A real, named attorney's face does more work
in this niche than in most: per `05-trust-signals.md`, it is the strongest
available counter to the "settlement mill" fear. Restrained motion and
disciplined yellow usage are deliberate departures from the niche's
common visual noise (`08-starter-template.md` flags overcrowded,
repetitive-CTA layouts as an anti-pattern); do not let hero generation
re-introduce that noise even if reference sites show it.
