# 10.1 - Build

Implements: Stage 10.1.

## Purpose

Clone the personal-injury-lawyers niche template, overlay the client's
brand-dna, build the site, and validate that all 10 pages, including all 7
case-type routes, render correctly.

## Inputs (per-client)

```
clients/[Client Name]/Pipeline Data/brand/brand-dna.json
templates/personal-injury-lawyers/
```

## Procedure

1. Run `tools/build-from-template.py`, which reads the active niche
   (`personal-injury-lawyers`) from `stack-state.json` and clones
   `templates/personal-injury-lawyers/` into
   `clients/[Client Name]/[Client Name] Website/`.
2. Fill `src/config/brand-dna.js` from `Pipeline Data/brand/brand-dna.json`.
3. Run the build (`npm run build` or equivalent).
4. Validate the route list against `09-sitemap.json`: Home, About, Practice
   Areas, Testimonials, FAQ, Contact, Blog, and the case-type routes for
   every case type present in `brand-dna.json`'s `services[]` /
   `pages.services.items[]` (up to 7: car-accident, truck-accident,
   motorcycle-accident, slip-and-fall, wrongful-death, dog-bite,
   brain-injury).
5. For each rendered case-type page, confirm the page is NOT empty: it has
   a non-empty H1 (matching the `"[Case Type] Lawyer in [City] |
   [Firm Name]"` pattern), a settlement figure or fallback, body text from
   `pages.services.items[slug].body`, and at least one FAQ item.
6. Confirm the LeadForm renders with exactly 4 fields (name, phone, email,
   case description) in every instance: hero, footer, case-type CTA, and
   contact page.
7. Confirm the footer renders `company.licenseNumber`,
   `hours.display`, and bar association links if present.
8. Confirm the header's "CALL NOW" link and "OPEN 24/7" badge (if
   applicable) render and remain visible/sticky on scroll.

## Pass criteria

- Build completes without errors.
- All sitemap routes resolve, including every case-type route for case
  types present in `brand-dna.json`. No case type listed in `services[]` is
  missing its `/case-types/{slug}` page.
- No case-type page renders with empty body, missing H1, or zero FAQ items.
- Every LeadForm instance renders exactly 4 fields.
- Footer renders license number and hours.
- Header CALL NOW link and OPEN 24/7 badge (if applicable) render and stay
  visible on scroll.

## When this stage halts

- Build fails (compile error, missing template). If
  `templates/personal-injury-lawyers/` does not exist, halt with a clear
  error pointing back to `/build-niche-template`.
- A case type present in `brand-dna.json`'s `services[]` has no
  corresponding `/case-types/{slug}` route, or that route renders empty.
- Any LeadForm instance renders with more or fewer than 4 fields.
- `company.licenseNumber` is missing from the rendered footer.

## Niche-specific notes

For this niche, the following pattern applies (derived from
`research/02-niche-research/personal-injury-lawyers/09-template-spec.md` +
the niche playbook):

The case-type pages are the build's most important validation target: this
template's primary SEO and conversion differentiator is shipping all
covered case types as fully realized pages with the complete trust stack
(no win/no fee restatement, single-figure StatsBar, explainer body, FAQ,
and repeated LeadForm), per `09-wireframe.md`'s CaseTypeLandingPage
composition. A build that compiles cleanly but produces a thin or empty
case-type page is a build failure for this niche even if no build-tool
error is raised.
