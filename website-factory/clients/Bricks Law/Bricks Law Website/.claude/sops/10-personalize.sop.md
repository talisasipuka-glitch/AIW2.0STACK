# 10.2 - Personalize

Implements: Stage 10.2.

## Purpose

Inject SEO metadata and schema markup into the built site: per-page titles
and meta descriptions, `Attorney`/`LegalService`/`LocalBusiness` JSON-LD,
`AggregateRating`/`Review` schema, `FAQPage` schema, `BreadcrumbList` on
case-type pages, and `sitemap.xml`.

## Inputs (per-client)

```
clients/[Client Name]/[Client Name] Website/dist/
clients/[Client Name]/Pipeline Data/seo/
clients/[Client Name]/Pipeline Data/brand/brand-dna.json
```

## Procedure

1. Run `inject-theme.mjs` (or equivalent) to write per-page `<title>` and
   meta description tags using the formulas confirmed at Stage 3:
   homepage = `"[Firm Name]: Personal Injury Lawyers [City/State]"`,
   case-type pages = `"[Case Type] Lawyer in [City] | [Firm Name]"`.
2. Inject `LegalService` and `Attorney` JSON-LD on the homepage and About
   page, populated from `brandDNA.company`, `brandDNA.team.founder`, and
   `brandDNA.address`. Include `company.licenseNumber` in the schema where
   the schema vocabulary supports it (e.g. as an `identifier` or in
   `description`).
3. Inject `LocalBusiness` JSON-LD with `brandDNA.address`,
   `brandDNA.contact`, and `brandDNA.hours.openingHours`. Only encode "Open
   24 hours" if `brandDNA.hours` confirms it; otherwise encode the actual
   hours.
4. Inject `AggregateRating` / `Review` schema from `brandDNA.reviews.rating`
   and `brandDNA.reviews.items[]` / `totalReviewCount`.
5. Inject `FAQPage` schema on the homepage FAQ section and on every
   case-type page's FAQ block, using `brandDNA.faq[]` and
   `brandDNA.pages.services.items[slug].faq[]` respectively.
6. Inject `BreadcrumbList` schema on all 7 (or fewer, per Stage 5)
   case-type pages: Home > Practice Areas > [Case Type].
7. Generate `sitemap.xml` covering all rendered routes, including every
   case-type route present in the build.
8. Validate all injected JSON-LD against schema.org structure (no malformed
   JSON, required properties present).

## Pass criteria

- Homepage and case-type pages carry the confirmed title/meta formulas.
- `LegalService`/`Attorney`/`LocalBusiness` JSON-LD present and valid on the
  homepage, with `company.licenseNumber` reflected.
- `AggregateRating`/`Review` schema present, matching `brandDNA.reviews`.
- `FAQPage` schema present on the homepage FAQ and every case-type page's
  FAQ block.
- `BreadcrumbList` schema present on every case-type page.
- `sitemap.xml` includes all rendered routes including all case-type
  routes.
- All JSON-LD validates (no malformed JSON, required fields present).

## When this stage halts

- Any injected JSON-LD is malformed or missing a required property for its
  schema type.
- `LocalBusiness` schema encodes "Open 24 hours" when
  `brandDNA.hours` does not confirm it. This is a factual-accuracy issue,
  not just a style issue, and must be corrected before deploy.
- A case-type page is missing its `FAQPage` or `BreadcrumbList` schema.

## Niche-specific notes

For this niche, the following pattern applies (derived from
`research/02-niche-research/personal-injury-lawyers/09-template-spec.md` +
the niche playbook):

This niche carries more schema types than most: `Attorney`, `LegalService`,
`LocalBusiness`, `AggregateRating`/`Review`, `FAQPage` (twice: homepage and
per case-type page), and `BreadcrumbList` (per case-type page). Per
`06-seo-landscape.md`, `FAQPage` schema is a direct response to the "People
Also Ask" content gap this niche's top organic results leave unaddressed,
and `BreadcrumbList` on case-type pages supports the
`"[case type] lawyer [city]"` ranking pattern. Treat the full schema set as
mandatory for every case-type page, not optional polish.
