# 11 - Deploy

Implements: Stage 11.

## Purpose

Deploy the validated build to Vercel production, using the standard
factory deploy procedure.

## Inputs (per-client)

```
clients/[Client Name]/[Client Name] Website/dist/
clients/[Client Name]/Pipeline Data/deploy/
```

## Procedure

1. Confirm all prior QA gates (10.4a design fidelity, 10.4b SOP QA, 10.4c
   build fidelity, 10.4d performance) have passed or have an explicit
   student override on record.
2. Run the standard Vercel deploy procedure per `vercel-deploy/SKILL.md`.
3. Confirm the production URL resolves and serves the homepage and all
   confirmed case-type routes.
4. Confirm `sitemap.xml` and `robots.txt` are served correctly at
   production URLs.
5. Confirm HTTPS is active and the production domain (if a custom domain is
   configured) resolves correctly.
6. Record the production URL and deploy timestamp in
   `Pipeline Data/deploy/`.

## Pass criteria

- Production URL is live and serves the homepage and all confirmed
  case-type routes with correct content.
- `sitemap.xml` and `robots.txt` accessible at production URLs.
- HTTPS active.
- Deploy record written.

## When this stage halts

- A prior QA gate has not passed and has no recorded student override.
- The production deploy fails or the production URL does not resolve after
  deploy.
- Any confirmed case-type route 404s on the production URL despite
  resolving correctly in the local build (a routing/rewrite configuration
  issue).

## Niche-specific notes

For this niche, the following pattern applies (derived from
`research/02-niche-research/personal-injury-lawyers/09-template-spec.md` +
the niche playbook):

Deploy procedure for this niche is standard with no template-specific
deploy steps. The one niche-relevant check worth a manual glance after
deploy: confirm the case-type routes resolve on the production domain with
the same correctness as in the local build, since these routes carry the
template's primary SEO value and a routing misconfiguration here would
silently undercut the case-type-page strategy.
