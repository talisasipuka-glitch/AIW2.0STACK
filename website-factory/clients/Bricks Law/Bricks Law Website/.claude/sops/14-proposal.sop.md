# 13 - Proposal

Implements: Stage 13. Runs after deploy.

## Purpose

Generate the final client-facing HTML proposal: a summary of the delivered
site, its competitive position, and next steps, in the serious,
professional, trustworthy tone this niche's clients expect from any vendor
they work with.

## Inputs (per-client)

```
clients/[Client Name]/Pipeline Data/delivery/
clients/[Client Name]/Pipeline Data/research/ (competitor data from Stage 2)
clients/[Client Name]/Pipeline Data/deploy/
```

## Procedure

1. Summarize the delivered site: pages built, the production URL, and the
   case-type pages shipped as the headline SEO feature.
2. Build the competitive argument using Stage 2's competitor research: if
   local competitors lack separate case-type landing pages (the common
   case per `06-seo-landscape.md`), state this plainly as the client's new
   advantage, with the specific keyword pattern
   (`"[case type] lawyer [city]"`) named.
3. Summarize the trust elements built into the site: the contingency
   guarantee placement, the settlement-figure StatsBar, the FounderStory,
   testimonials, and the full schema markup set (Attorney, LegalService,
   LocalBusiness, FAQPage, BreadcrumbList).
4. List any follow-up items from Stage 12's delivery report (e.g. real
   attorney photo still needed) as "recommended next steps," framed
   constructively, not as a deficiency.
5. Generate the HTML proposal using `proposal-writing/SKILL.md`'s voice and
   structure standards.
6. Do not include any pricing, contract terms, or guarantees beyond what
   the student has explicitly approved for this client.

## Pass criteria

- Proposal summarizes delivered pages, production URL, and the case-type
  page advantage with a specific keyword pattern cited.
- Competitive argument is grounded in Stage 2's actual competitor data, not
  generic claims.
- Trust elements built into the site are summarized accurately (matching
  what Stage 10.2/10.4b actually verified, not aspirational claims).
- Follow-up items framed as recommended next steps.
- No unapproved pricing or contract terms included.
- Copy-lint passes (zero em-dashes, no blocklist hits).

## When this stage halts

- Stage 11 (deploy) has not completed, so no production URL exists.
- The competitive argument cannot be grounded in real Stage 2 data (e.g.
  Stage 2 found no usable competitor data). In this case, omit the
  competitive comparison rather than inventing one.
