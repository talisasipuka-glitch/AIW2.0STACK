# 10.4b - SOP QA

Implements: Stage 10.4b.

## Purpose

Score the built site against the universal SOP layer plus this niche's
per-niche compliance checklist, covering composition, copy, schema, and
the niche's specific CRO deviations (4-field form cap, trust claim
placement).

## Inputs (per-client)

```
clients/[Client Name]/[Client Name] Website/dist/
templates/personal-injury-lawyers/.claude/checklists/sop-compliance.md
.claude/checklists/*.universal.md (when present)
```

## Procedure

1. Run `tools/copy-lint.py --check` against the rendered copy. Zero
   em-dashes, no `ai-vocab-blocklist.md` hits, curly quotes,
   `tabular-nums` on dollar figures.
2. Check `composition.section_order` on the homepage matches
   `09-wireframe.md`: Hero, ProcessSteps, StatsBar, PressBand, WhyChooseUs,
   PracticeAreasGrid, FounderStory, Testimonials, FAQSection, CTABand.
3. Check `composition.above_fold`: Hero + sticky Header end above the 900px
   desktop fold; LeadForm's submit button is visible without scrolling at
   1440x900.
4. Check `composition.case_type_pages.all_seven_present` (or fewer, per
   Stage 5's confirmed sitemap): every confirmed case-type route resolves
   to a rendered page with a non-empty
   `pages.services.items[slug]` entry.
5. Check `composition.lead_form.field_count`: every `LeadForm` instance
   (hero, footer, case-type CTA, contact page) renders exactly 4 fields
   (name, phone, email, case description). This is the niche's CRO
   deviation from the 5 to 7+ field norm; flag ANY instance with more or
   fewer than 4.
6. Check `composition.header.sticky_cta`: the header's "CALL NOW" phone
   link remains visible at all scroll positions on desktop and mobile.
7. Check trust-claim placement: the contingency guarantee ("no win, no
   fee" or client equivalent) appears near the hero CTA, footer CTA, every
   case-type page's RiskRemovalBand, and the final CTABand.
8. Check `company.licenseNumber` renders in the footer on every page.
9. Check schema presence per Stage 10.2: `LegalService`/`Attorney`/
   `LocalBusiness` on homepage, `FAQPage` on homepage and every case-type
   page, `BreadcrumbList` on every case-type page,
   `AggregateRating`/`Review` matching `brandDNA.reviews`.
10. Cross-reference against `.claude/checklists/sop-compliance.md` for any
    additional niche-specific items generated at Module 2D Phase 10.

## Pass criteria

- Copy-lint passes with zero findings.
- Section order matches `09-wireframe.md` exactly.
- Above-fold composition check passes.
- All confirmed case-type pages present and non-empty.
- Every LeadForm instance has exactly 4 fields.
- Header sticky CTA check passes.
- Trust-claim placement check passes at all 4 locations (hero CTA, footer
  CTA, case-type RiskRemovalBand, final CTABand).
- License number renders in footer on every page.
- Full schema set present per Stage 10.2.
- `.claude/checklists/sop-compliance.md` items all pass.

## When this stage halts

- Copy-lint fails. Route back to Stage 6/Stage 10.3 for correction.
- Any LeadForm instance has more or fewer than 4 fields. This is a hard
  CRO-deviation requirement for this niche, not a style preference.
- A confirmed case-type page is missing or empty.
- `company.licenseNumber` missing from the footer on any page.
- Required schema (per item 9) missing on any applicable page.

## Niche-specific notes

For this niche, the following pattern applies (derived from
`research/02-niche-research/personal-injury-lawyers/09-template-spec.md` +
the niche playbook):

The 4-field LeadForm cap is the niche's deliberate CRO deviation from the
norm (`pick.json.notedDeviation`, `04-cro-patterns.md`): every crawled
competitor runs 5 to 7+ fields. This SOP QA gate is the last automated
checkpoint before deploy that can catch a regression back toward more
fields (e.g. if Stage 10.3 or a copy edit re-introduces a "case type"
dropdown into the primary form). Treat any field-count violation as a hard
fail regardless of how minor it looks. Reference
`.claude/checklists/sop-compliance.md` (Module 2D Phase 10 output) for the
complete per-niche checklist this stage scores against.
