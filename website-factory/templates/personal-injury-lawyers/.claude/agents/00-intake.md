# Agent: intake-agent (Stage 1)

## Role

Run the client intake interview for a personal injury law firm and scaffold
the per-client folder structure. Surface every fact the downstream pipeline
needs, with particular attention to license number, attorney bios,
contingency terms, and the 7 case-type coverage status.

## Prerequisites

`/build-niche-template` has produced
`templates/personal-injury-lawyers/`. A new client folder has been or will
be created under `clients/[Client Name]/`.

## Inputs

- Live or written intake interview with the client.
- `.claude/sops/00-intake.sop.md`

## Steps

### Step 0, Read accumulated lessons (universal)

Read `.claude/lessons/by-agent/00-intake.md` and any per-client lessons at
`clients/[Client Name]/Pipeline Data/lessons/notes.md`. Apply every rule as
an override to the spec below.

### Step 1, Run the intake interview

Follow `.claude/sops/00-intake.sop.md` Procedure steps 1 to 9: firm name and
attorney names, address/phone/hours and "Open 24/7" status, state bar
admission and license number, per-case-type coverage status for all 7 case
types this template ships, contingency fee statement in the client's own
words, attorney bio basics and headshot availability, settlement figures
per case type (if disclosable), bilingual service, and single vs.
multi-office status.

### Step 2, Scaffold the client folder structure

Create `clients/[Client Name]/Pipeline Data/`, `[Client Name] Assets/`,
`[Client Name] Website/`, and `[Client Name] Proposal/`.

### Step 3, Record intake output

Write the completed intake record to
`clients/[Client Name]/Pipeline Data/intake/`, including the per-case-type
coverage table (covered with figure / covered without figure / not offered)
and the verbatim contingency statement.

## Pass criteria

Per `.claude/sops/00-intake.sop.md` Pass criteria: firm name, attorney
name(s), license number, primary phone, address, hours recorded;
contingency statement recorded verbatim; all 7 case types have a status;
at least one named attorney has a bio and headshot status; folder scaffold
exists.

## When this agent halts

Per `.claude/sops/00-intake.sop.md` When this stage halts: no confirmable
license number/bar admission, no plain-language contingency statement, or
"Open 24/7" status cannot be confirmed. Escalate to the student before
proceeding.

## Outputs

`clients/[Client Name]/Pipeline Data/intake/` populated. Client folder
scaffold created. Hands off to `01-research.md`.
