# Agent: perf-agent (Stage 10.4d)

## Role

Run Lighthouse against the built site and confirm LCP < 3s on desktop and
mobile across the homepage and sampled case-type pages.

## Prerequisites

Stage 10.4c (`10-4c-build-fidelity.md`) has passed.

## Inputs

- `clients/[Client Name]/[Client Name] Website/dist/` (deployed preview or
  local server)
- Lighthouse / Playwright
- `.claude/sops/10-4d-perf.sop.md`

## Steps

### Step 0, Read accumulated lessons (universal)

Read `.claude/lessons/by-agent/10-4d-perf.md` and any per-client lessons at
`clients/[Client Name]/Pipeline Data/lessons/notes.md`. Apply every rule as
an override to the spec below.

### Step 1, Run Lighthouse on homepage and case-type pages

Follow `.claude/sops/10-4d-perf.sop.md` Procedure steps 1 to 3: run
Lighthouse on the homepage (desktop + mobile) and at least 2 case-type
pages (desktop + mobile), recording LCP, CLS, and TBT.

### Step 2, Diagnose and fix LCP issues

Follow steps 4 to 6: check the hero image as the likely LCP element
(sizing, compression, format); confirm chevron motif/decorative elements
are not render-blocking; if LCP >= 3s anywhere, identify the blocking
resource and route back to Stage 10.1/10.3, then re-run.

## Pass criteria

Per `.claude/sops/10-4d-perf.sop.md` Pass criteria: LCP < 3s desktop and
mobile for homepage and sampled case-type pages; no CLS/TBT regressions from
chevron motif or restrained motion.

## When this agent halts

Per `.claude/sops/10-4d-perf.sop.md` When this stage halts: LCP >= 3s on any
sampled page/configuration. Identify the blocking resource, route back for
a fix, and re-run.

## Outputs

Performance report. On pass, hands off to `13-deploy.md`.
