# Agent: deploy-agent (Stage 11)

## Role

Deploy the validated build to Vercel production using the standard factory
deploy procedure.

## Prerequisites

Stages 10.4a to 10.4d have passed or have an explicit student override on
record.

## Inputs

- `clients/[Client Name]/[Client Name] Website/dist/`
- `clients/[Client Name]/Pipeline Data/deploy/`
- `vercel-deploy/SKILL.md`
- `.claude/sops/13-deploy.sop.md`

## Steps

### Step 0, Read accumulated lessons (universal)

Read `.claude/lessons/by-agent/13-deploy.md` and any per-client lessons at
`clients/[Client Name]/Pipeline Data/lessons/notes.md`. Apply every rule as
an override to the spec below.

### Step 1, Confirm gates and deploy

Follow `.claude/sops/13-deploy.sop.md` Procedure steps 1 to 2: confirm all
QA gates passed or have a recorded override; run the standard Vercel deploy
procedure.

### Step 2, Verify production

Follow steps 3 to 6: confirm the production URL serves the homepage and all
confirmed case-type routes; confirm `sitemap.xml` and `robots.txt` are
served; confirm HTTPS and custom domain (if configured); record the
production URL and deploy timestamp.

## Pass criteria

Per `.claude/sops/13-deploy.sop.md` Pass criteria: production URL live and
serving homepage + all confirmed case-type routes; `sitemap.xml`/
`robots.txt` accessible; HTTPS active; deploy record written.

## When this agent halts

Per `.claude/sops/13-deploy.sop.md` When this stage halts: a prior QA gate
has not passed with no recorded override; production deploy fails or URL
does not resolve; a confirmed case-type route 404s on production despite
working locally.

## Outputs

`clients/[Client Name]/Pipeline Data/deploy/` with production URL and
timestamp. Hands off to `12-delivery.md`.
