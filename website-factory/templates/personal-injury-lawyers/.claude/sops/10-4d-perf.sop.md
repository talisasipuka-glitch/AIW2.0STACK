# 10.4d - Performance

Implements: Stage 10.4d.

## Purpose

Run Lighthouse against the built site and confirm LCP (Largest Contentful
Paint) is under 3 seconds on both desktop and mobile, across the homepage
and case-type pages.

## Inputs (per-client)

```
clients/[Client Name]/[Client Name] Website/dist/ (deployed preview or local server)
```

## Procedure

1. Run Lighthouse against the homepage in both desktop and mobile
   configurations.
2. Run Lighthouse against at least 2 case-type pages (the highest-traffic
   ones per Stage 3's keyword research, if known, otherwise the first 2 in
   sitemap order) in both configurations.
3. Record LCP, CLS, and TBT for each page/configuration.
4. Pay particular attention to the hero image (Stage 9 output) as the
   likely LCP element on the homepage: confirm it is appropriately sized,
   compressed, and uses modern formats (WebP/AVIF) with fallback.
5. Confirm the chevron motif overlays and any decorative elements (Stage
   10.3) are not render-blocking and do not delay LCP.
6. If LCP exceeds 3s on any page/configuration, identify the specific
   blocking resource (unoptimized image, blocking font load, render-
   blocking script) and route back to Stage 10.1/10.3 for a fix.

## Pass criteria

- LCP < 3s on desktop and mobile for the homepage.
- LCP < 3s on desktop and mobile for the sampled case-type pages.
- No CLS or TBT regressions introduced by the chevron motif or restrained
  motion animations.

## When this stage halts

- LCP >= 3s on any sampled page/configuration. Identify the blocking
  resource and route back for a fix; re-run this stage after the fix.

## Niche-specific notes

For this niche, the following pattern applies (derived from
`research/02-niche-research/personal-injury-lawyers/09-template-spec.md` +
the niche playbook):

The standard < 3s LCP gate applies to this niche without modification.
Given the hero's split layout (attorney photo + LeadForm) is the
highest-weighted SSIM region (0.95/0.20) and the most likely LCP element,
verify the attorney photo specifically is optimized; a heavy, unoptimized
hero photo is the most probable cause of an LCP failure in this template.
Case-type pages, being the primary SEO entry points per
`06-seo-landscape.md`, should be sampled for performance just as
rigorously as the homepage, since a slow case-type page directly undermines
the SEO advantage these pages are built to capture.
