# 7 - Brand DNA Extraction

Implements: Stage 7.

## Purpose

Extract the client's brand DNA into the canonical 32-key shape contract
(`references/brand-dna.shape.js`), populating `brand-dna.json` with this
niche's specific fields: license number, attorney bios, settlement figures
per case type, and the trust-claim copy locked at Stage 6.

## Inputs (per-client)

```
clients/[Client Name]/Pipeline Data/intake/
clients/[Client Name]/Pipeline Data/research/
clients/[Client Name]/Pipeline Data/copy/
clients/[Client Name]/[Client Name] Assets/
```

## Procedure

1. Populate `company.name`, `company.shortName`, and `company.licenseNumber`
   from Stage 1. `licenseNumber` is a hard requirement for this niche;
   verify it is non-empty before proceeding.
2. Populate `contact.phone`, `contact.phoneTelLink`, `address.*`, and
   `hours.*` (including `hours.emergencyBadge` and `hours.display`) from
   Stage 1, only marking 24/7 availability if confirmed.
3. Populate `team.founder` and any `team_members[]` with the named
   attorney(s): name, title, headshot path (from Stage 4 assets), years
   practicing, and bio summary.
4. Populate `services[]` with the covered case types from Stage 5's
   confirmed sitemap (the case types this client actually practices, in the
   order they should appear in PracticeAreasGrid).
5. Populate `pages.services.items[]` for each covered case type with:
   `name`, `slug`, `description`, `body`, `faq[]` (from Stage 6 copy), and
   `settlementFigure` (niche-specific extension field within the canonical
   `pages.services` bucket, per `09-template-spec.md` Section 4, item 4).
   Where a case type has no disclosable figure, omit `settlementFigure`
   rather than inventing one.
6. Populate `press_logos[]` (if the client has real press mentions from
   Stage 2/4) or leave empty so the PressBand falls back to
   `trust_badges[]`. Populate `trust_badges[]` with bar association,
   membership, and local award badges from Stage 4.
7. Populate `why_choose_us[]`, `process_steps[]`, `reviews.items[]`,
   `reviews.rating`, `faq[]` (homepage), and all `copy.*` paths from Stage
   6's copy deck verbatim. Do not paraphrase or shorten locked copy.
8. Populate `location_pages[]` per Stage 5's decision (empty for
   single-office clients).
9. Compute an extraction confidence score. Required fields for this niche
   (in addition to the universal required set):
   `company.licenseNumber`, `team.founder`, at least one
   `pages.services.items[]` entry with a non-empty `body`.
10. If confidence < 0.70, halt at the approval gate per the factory's
    universal Stage 7 process (`/approve-brand-dna`).

## Pass criteria

- `brand-dna.json` validates against `references/brand-dna.shape.js`.
- `company.licenseNumber` is non-empty.
- `team.founder` is populated with name, bio, and headshot path.
- Every case type in `services[]` has a matching, fully populated entry in
  `pages.services.items[]` (name, slug, description, body, faq[]).
- `settlementFigure` is present only where Stage 1/5 confirmed a
  disclosable figure.
- All `copy.*` fields carry Stage 6's locked copy without paraphrase.

## When this stage halts

- Extraction confidence < 0.70: triggers the approval gate, per the
  factory's universal Stage 7 process.
- `company.licenseNumber` is missing or empty: hard halt regardless of
  overall confidence score, since this niche's `LegalService`/`Attorney`
  schema (Stage 10.2) and footer trust block both require it.
- No `team.founder` entry exists: hard halt, since the FounderStory section
  has no content without it and this is the niche's strongest trust
  element.

## Niche-specific notes

For this niche, the following pattern applies (derived from
`research/02-niche-research/personal-injury-lawyers/09-template-spec.md` +
the niche playbook):

`pages.services.items[]` carries a niche-specific extension field,
`settlementFigure`, used by StatsBar (homepage, multi-figure) and StatsBar
single-figure variant (case-type pages). This field is optional per item;
absence means the StatsBar section either omits that case type's figure or
(on the homepage) falls back to an aggregate recovery figure decided at
Stage 5. `team.founder` and any additional `team_members[]` should carry
enough bio detail to support both the FounderStory section (2-paragraph
narrative) and the About page's expanded FounderStory variant.
